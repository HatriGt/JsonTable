# Product

## Register

product

## Users

Developers, data analysts, and support engineers who inspect API responses, config files, and log payloads. They work at a desk under normal office lighting, often mid-debug with a clipboard full of JSON and a specific field to find in under a minute.

## Product Purpose

JSON-Table turns nested JSON into a navigable tree and a filterable grid so users can locate, compare, and edit values without mentally parsing braces. Success means opening a file and reaching the target cell in seconds, with zero data leaving the browser.

## Brand Personality

Precise, fast, trustworthy. Terminal-native clarity with spreadsheet familiarity — not playful, not corporate-generic.

## Anti-references

Gradient headline text, glassmorphism cards, orb empty states, identical icon-card feature grids, cream SaaS landing pages, decorative motion that delays the task, upload-to-cloud patterns.

## Design Principles

1. **Task first** — Every screen earns its pixels by shortening time-to-value.
2. **Show the product** — The live grid demo is the hero; marketing copy supports it.
3. **Local by default** — Privacy and speed are product features, stated once, not repeated as decoration.
4. **Monospace for data** — Keys, paths, and grid chrome use mono; UI chrome stays clean sans.
5. **One identity** — Landing and workspace share tokens, tone, and density language.

## Accessibility & Inclusion

WCAG 2.1 AA minimum. Respect `prefers-reduced-motion`. Body text contrast ≥4.5:1. Keyboard paths for paste, open, and navigation.
